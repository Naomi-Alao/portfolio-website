Product Overview

A simplified version of my portfolio website to showcase my recent UX and UI work to share with my professional network, potential clients, recruiters, and hiring managers. My portfolio website is responsive and coded from scratch with HTML, CSS and JavaScript.

Context

Understanding the foundations of programming is an important asset for a designer. Especially for designers who regularly collaborate with developers. In this project, I gained valuable insights into how to work with design considerations alongside technical requirements.

Key-takeaway

Understanding what developers need from designers when exporting assets and deliverables as it pertains to design systems and lables E.G: labelling states: hover, click, visited, focus, active. Labelling styling: padding, border, flex-direction etc. This understanding helps me as a designer create designs that are technically feasible and easily understood by developers.

Challenge

In this course, I learned the basics of frontend development, covering programming languages (HTML JavaScript and CSS). With a bit of prior coding knowledge, I built a responsive portfolio website from scratch to showcase my UX/UI focused projects and FrontEnd Development skills.

Goals

  • Create a professional digital presence the showcases my design background, UX/UI projects, relevant work experience, and design process.
  • Create a responsive site that works across all modern browsers and devices (mobile, tablet, desktop)
  • Write clean, validated code with no errors
  • Host in a Git repository on GitHub
  • To create a distinct design that makes me stand out as a candidate for a UX/UI opening

Design Process

I began my project by creating wireframes and writing all the copy that would be featured in my portfolio. By starting with the content I was able to design my website around my work. With all of my content prepared I was able to begin creating a mobile first approach website, starting with the HTML.

By starting with this approach, I was able to structure the design around my website. I began creating my website with a mobile first approach, starting with the HTML.

    • Cross Browsing Testing Highlights

    To test whether my website works properly in all necessary browsers and on all devices. I conducted a type of quality assurance called cross-browser testing.

    As browsers support different levels of features and due to industry standards evolve very quickly, some browsers can’t implement new features at the same speed. This is why it’s so important to know which features are supported in which browsers

    For instance in my case, Yandex a Windows browser was the only browser to not display the emoji's on my website

    As a tool, I used BrowserStack live cloud platform to. This is a tool which provides developer teams to comprehensively test their websites and mobile applications for functionality, performance, and visual appeal so they can release bug-free software faster and at scale.

    User Test Highlights

    5 Participants who have a professional background in Tech, Design and/or Recruiting were chosen for the usability testing study. Participant ages range from 30-65 who have been selected from my personal network. Testing session were either moderated in person or remote and took 20-30 minutes to complete. Participants were asked background questions, and completed six short tasks to test the site’s navigation and overall ease of usability. Using the Jakob Nielsen’s error rating scale, I created a Test Report of issues and prioritized by severity.

    User Test Highlights

    All 5 participants have encountered various levels of issues while completing the 5 scenario tasks. The top test errors are listed below in order of severity. Issue 1: Users didn’t find the view case study button to be attention grabbing (Error Rating: 3) Suggested Change: Used a different colour to make the button stand out more Evidence: 5/5 participants mentioned that there should be a change to the button

    Issue 2: Users didn’t think the resume matched the theme of the website (Error Rating: 3) Suggested Change: Change the colour palate to match the website Evidence: 2/5 participants suggested changing the theme of the resume

    Issue 3: Text looked too blown out and spread too far (Error Rating: 2) Suggested Change: Break up big blocks of text and condense full page width text sections Evidence: 2/5 participants

    Issue 4: Users also checked contacts' page for email and LinkedIn links (Error Rating: 2) Suggested Change: Adding LinkedIn and Email address to the contacts' page once if has been created Evidence: 3/5 participants checked Contacts page before navigating to the footer with the necessarily links

    Issue 5: Add more photos to the About section and hero page (Error Rating: 1) Suggested Change: Adding images to the about and hero page Evidence: 2/5 participants mention the addition of more Photos
